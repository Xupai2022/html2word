## 🔧 问题已修复！

之前页脚位置设置不生效的问题已经解决。现在 `FOOTER_DISTANCE_FROM_BOTTOM` 配置会正确应用。

## 当前设置

您的页脚当前配置为距离页面底部 **0.1 英寸**（非常低）。

## 如何调整页脚位置

### 方法1：直接编辑配置文件

编辑 `src/html2word/config/header_footer_config.py` 文件，找到第 161 行：

```python
# 页脚距离页面底部的距离（英寸）
FOOTER_DISTANCE_FROM_BOTTOM = 0.3  # inches
```

**调整建议：**
- **更低（更靠近底部）**：改为 `0.2` 或 `0.15`
- **标准位置**：使用 `0.5`（Word默认）
- **更高（远离底部）**：改为 `0.7` 或 `1.0`

### 方法2：在代码中动态调整

```python
from html2word.converter import HTML2WordConverter

converter = HTML2WordConverter()

# 动态调整页脚位置
converter.document_builder.configure_header_footer(
    FOOTER_DISTANCE_FROM_BOTTOM=0.2  # 设置为0.2英寸，让页脚更低
)

converter.convert_file("input.html", "output.docx")
```

## 相关配置参数

除了页脚位置，您还可以调整：

```python
# 页眉距离页面顶部的距离
HEADER_DISTANCE_FROM_TOP = 0.5  # inches

# 页边距设置（也会影响页眉页脚的可用空间）
TOP_MARGIN = 1.0      # 顶部边距
BOTTOM_MARGIN = 1.0   # 底部边距（影响页脚最低位置）
LEFT_MARGIN = 1.0     # 左边距
RIGHT_MARGIN = 1.0    # 右边距

# 页脚内容之间的间距
FOOTER_INTERNAL_SPACING = 6  # pt (磅)
```

## ⚠️ 重要说明

### 页脚位置的两个关键配置

1. **`FOOTER_DISTANCE_FROM_BOTTOM`**：页脚距离页面底部边缘的距离
2. **`BOTTOM_MARGIN`**：页面底部边距（默认 1.0 英寸）

### 关系说明

- 页脚实际位置 = 页面底部边缘 + `FOOTER_DISTANCE_FROM_BOTTOM`
- 如果 `FOOTER_DISTANCE_FROM_BOTTOM` = 0.1，页脚将非常贴近页面底部
- `BOTTOM_MARGIN` 影响正文内容的底部边界，不是页脚位置的限制

### 建议配置

如果您想让页脚尽可能低：
```python
FOOTER_DISTANCE_FROM_BOTTOM = 0.1  # 页脚非常低
BOTTOM_MARGIN = 0.8                # 减小底部边距（可选）
```

## 注意事项

1. **最小值限制**：建议不要低于 0.1 英寸，否则可能超出打印机边距
2. **打印机边距**：某些打印机有最小边距要求，通常为 0.2-0.25 英寸
3. **不需要调整 `BOTTOM_MARGIN`**：除非您也想调整正文内容的底部边界

## 示例对比

| 设置值 | 效果描述 |
|--------|----------|
| 0.15 inches | 非常低，几乎贴底 |
| 0.2 inches | 较低，适合紧凑布局 |
| **0.3 inches** | **当前设置** |
| 0.5 inches | Word 默认位置 |
| 0.7 inches | 较高，留出更多底部空间 |
